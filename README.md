# AutoTrader — 한국투자증권 자동매매 프로그램

Python + KIS API 기반 주식 자동매매 프로그램입니다.  
웹 UI를 통해 전략 전환, 실시간 잔고 확인, 로그 모니터링을 할 수 있습니다.

## 3. API 키 설정

`.env.example` 파일을 복사해서 `.env` 파일을 만들고 실제 값을 입력합니다.

```bash
# .env 파일 생성
cp .env.example .env
```

```dotenv
# .env
KIS_ID=본인_HTS_로그인_아이디
KIS_ACCOUNT=계좌번호-01          # 예: 50012345-01

KIS_APPKEY=발급받은_AppKey_36자리
KIS_SECRETKEY=발급받은_SecretKey_180자리
```

### API 키 발급 순서

1. https://apiportal.koreainvestment.com 접속 후 로그인
2. **My Page → 앱 등록** 클릭
3. 앱 이름 입력 → **실전투자** 선택 → 등록
4. 생성된 **AppKey**, **AppSecret** 복사 후 `.env`에 붙여넣기

> AppKey 유효기간은 **90일**입니다. 만료 전 KIS Developers에서 갱신하세요.

## 4. 설정 파일 안내

모든 설정은 `config.yaml` 파일 하나에서 관리합니다.

```yaml
# 전략 선택 (UI에서도 변경 가능)
strategy: rsi   # ma_cross | rsi | bollinger | volatility_breakout

# 투자 모드
virtual: false  # true: 모의투자 / false: 실전투자

# 매매 주기 (분)
interval_minutes: 5

# 조건 검색
screener:
  min_volume: 500000    # 최소 거래량
  min_price: 5000       # 최소 주가 (원)
  max_price: 200000     # 최대 주가 (원)
  max_stocks: 5         # 최대 선택 종목 수
  watchlist:            # 감시할 종목 코드
    - "005930"          # 삼성전자
    - "000660"          # SK하이닉스
    - "035420"          # NAVER

# 리스크 관리
risk:
  max_buy_amount: 500000  # 1회 최대 매수금액 (원)
  stop_loss_pct: -5.0     # 손절 기준 (%)
  take_profit_pct: 10.0   # 익절 기준 (%)

# 전략 파라미터
rsi:
  period: 14
  oversold: 30
  overbought: 70
```

## 5. 실행 방법

### 방법 A — UI 포함 실행 (권장)

```bash
python app.py
```

브라우저에서 http://localhost:5000 접속 후 **시작하기** 버튼 클릭.

### 방법 B — UI 없이 터미널에서 실행

```bash
python main.py
```

`yes` 입력 후 시작. 로그는 터미널과 `trading.log` 파일에 기록됩니다.

## 6. UI 사용 방법

### 화면 구성

```
┌─────────────────────────────────────────────┐
│  AutoTrader   [● 실행 중]   실전투자  [중지하기] │  ← 헤더
├─────────────────────────────────────────────┤
│  계좌 현황                                    │
│  [현금 잔고]  [주식 평가금액]  [총 자산]        │  ← 잔고 카드
├─────────────────────────────────────────────┤
│  보유종목  (2)                                │
│  종목코드 | 수량 | 평균단가 | 현재가 | 수익률    │  ← 보유종목 테이블
├─────────────────────────────────────────────┤
│  전략 설정  (dark 섹션)                       │
│  [RSI] [이동평균 크로스] [볼린저밴드] [변동성돌파]│  ← 전략 선택
├─────────────────────────────────────────────┤
│  실행 로그                    [초기화]         │
│  10:30:01  INFO  매매 사이클 시작              │  ← 실시간 로그
└─────────────────────────────────────────────┘
```

### 봇 시작/중지

- 헤더 우측 **시작하기** 버튼 클릭 → 자동매매 시작
- 실행 중에는 버튼이 **빨간색 "중지하기"** 로 바뀜
- 다시 클릭하면 중지

### 전략 변경

1. **봇이 중지된 상태**에서 전략 섹션의 pill 버튼 클릭
2. 선택한 전략이 파란색으로 활성화
3. `config.yaml`에 자동 저장됨
4. 봇을 다시 시작하면 새 전략으로 실행

> 봇이 실행 중일 때 전략 버튼을 누르면 경고 메시지가 표시됩니다.

### 잔고 새로고침

- 잔고는 **60초마다** 자동 갱신
- 헤더 옆 **새로고침(↺) 버튼** 클릭 시 즉시 갱신

### 자동 갱신 주기

| 항목 | 주기 |
|------|------|
| 봇 상태 | 3초 |
| 실행 로그 | 5초 |
| 잔고 / 보유종목 | 60초 |

## 7. 전략 설명

### RSI (상대강도지수)

- RSI가 **30 이하** 과매도 구간에서 반등 → 매수
- RSI가 **70 이상** 과매수 구간에서 하락 → 매도
- 횡보장에서 효과적

```yaml
rsi:
  period: 14      # RSI 계산 기간 (기본 14)
  oversold: 30    # 매수 기준값
  overbought: 70  # 매도 기준값
```

### 이동평균 크로스오버 (MA Cross)

- 단기 이평선이 장기 이평선을 **위로 돌파** → 매수 (골든크로스)
- 단기 이평선이 장기 이평선을 **아래로 돌파** → 매도 (데드크로스)
- 추세 추종형 전략

```yaml
ma_cross:
  short_period: 5   # 단기 이동평균 기간
  long_period: 20   # 장기 이동평균 기간
```

### 볼린저밴드 (Bollinger Bands)

- 주가가 하단 밴드 이탈 후 **회복** → 매수
- 주가가 상단 밴드 이탈 후 **하락** → 매도
- 변동성이 큰 종목에 효과적

```yaml
bollinger:
  period: 20      # 기간
  std_dev: 2.0    # 표준편차 배수 (밴드 폭)
```

### 변동성 돌파 (Volatility Breakout)

- 매수가 = 당일 시가 + (전일 고가 - 전일 저가) × k
- 현재가가 매수가를 돌파하면 → 매수
- 당일 매수, 장 마감 전 매도
- 구현이 단순하고 실전에서 검증된 전략

```yaml
volatility_breakout:
  k: 0.5   # 변동성 계수 (0.3 ~ 0.7 범위 추천)
```

## 8. 매매 조건 검색 설정

`config.yaml`의 `screener` 섹션에서 감시 종목과 필터 조건을 설정합니다.

```yaml
screener:
  min_volume: 500000    # 이 거래량 미만 종목은 제외
  min_price: 5000       # 이 가격 미만 종목은 제외 (저가주 필터)
  max_price: 200000     # 이 가격 초과 종목은 제외
  max_stocks: 5         # 최대 몇 개 종목에 진입할지
  watchlist:
    - "005930"          # 감시할 종목 코드 추가/제거
    - "000660"
```

**종목 코드 찾는 방법**: 네이버 증권, 한국거래소(KRX) 등에서 종목명으로 검색.

## 9. 리스크 관리 설정

```yaml
risk:
  max_buy_amount: 500000   # 종목당 최대 매수 금액
  stop_loss_pct: -5.0      # -5% 손실 시 자동 손절
  take_profit_pct: 10.0    # +10% 수익 시 자동 익절
```

- `max_buy_amount`를 낮게 설정할수록 분산 투자 효과
- 손절/익절은 매 사이클마다 보유 종목에 대해 자동 체크

## 11. 주의사항

> **실전투자는 실제 금전적 손실이 발생할 수 있습니다.**

- 처음 사용 시 반드시 소액으로 테스트하세요.
- `max_buy_amount`를 낮게 설정해 리스크를 제한하세요.
- API 키는 절대 타인과 공유하거나 GitHub 등에 올리지 마세요 (`.gitignore`에 `.env` 포함됨).
- AppKey는 **90일마다 갱신**이 필요합니다.
- KIS API는 **1초당 20건, 1일 10만건** 호출 제한이 있습니다.
- 장 운영 시간(평일 09:00 ~ 15:30) 외에는 주문이 실행되지 않습니다.
- 프로그램을 종료하면 봇도 함께 중지됩니다. 장시간 운용 시 서버 또는 항상 켜둔 PC에서 실행하세요.
